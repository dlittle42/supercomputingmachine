Speaking in color: A Web Speech API experiment

Week 14, 2014

It’s no secret that mobile is changing the way we think about, design and interact with the web. Touch has replaced the point-and-click model, if not in practice, in spirit. From typography to fluid grids to button sizes, even if you’re sitting at your desktop you feel its effects. Yet even as we designers and developers grapple with what that means for ourselves and our clients, the next disruption is already beginning.

Google Voice and Apple’s Siri have brought speech recognition to the mainstream and recently Google introduced a javascript Web Speech API in the latest release of Chrome. As of Chrome version 25, developers can easily convert audio to text using this API, finally making voice a viable method of interaction. How might this change how we think about interaction design? What are the possibilities for improved accessibility? And will applications even need a UI as we’ve come to know them?

All big questions worth exploring, but for starters, we’ve put together this little experiment to see what some of the possibilities might be. In this demo, once you activate your microphone you can speak the names of different colors and the screen background will change to match (only CSS color names, mind you). And for some good old-fashioned audio feedback, we’ve incorporated a separate text-to-speech library, speak.js. Try it out and let us know how you feel about the possibilities of speech recognition.
